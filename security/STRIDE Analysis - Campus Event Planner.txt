# STRIDE Analysis — Campus Event Planner

| CATEGORY | THREAT | SEVERITY | CONCRETE EXAMPLE | AFFECTED ENDPOINTS | FIX |
|----------|--------|----------|------------------|--------------------|-----|
| Spoofing | No authentication on any endpoint — anyone can create events, delete registrations, or access the admin panel with no identity check | Critical | `POST /events` with any title/date creates a legitimate-looking event; `GET /admin` accessed by any anonymous user | `POST /events` `PATCH /events/{id}` `GET /admin` | C026 |
| Tampering | Any user can overwrite any event's fields with no ownership check; JSON keys are injected as column names in the UPDATE query | Critical | `PATCH /events/1` with `{"capacity": 0}` silently zeroes out an event; crafted key `{"capacity=0 WHERE 1=1 --": "x"}` wipes every row in the table | `PATCH /events/{id}` | C025 |
| Repudiation | No logging on any write operation — fraudulent or destructive actions leave no audit trail | Medium | Attacker creates fake events, deletes all registrations, then removes the events — no IP, timestamp, or identity is ever recorded | `POST /events` `PATCH /events/{id}` `DELETE /events/{id}/registrations` | C029 |
| Info. disclosure | `debug=True` exposes Werkzeug interactive console; SQL errors from injection leak table and column names; `X-Powered-By` header fingerprints the stack; hardcoded secrets in source | Critical | Malformed `?q=` parameter triggers a MySQL syntax error — response includes table name and query structure; visiting any error URL shows full Python source in the browser | `GET /events/search` `GET /admin` `PATCH /events/{id}` | C027 |
| Denial of service | No rate limiting; `GET /events/search` runs a full-table `LIKE` scan on every request with no index; `PATCH /events/{id}` crashes on missing JSON body | High | 50 threads flood `POST /events` and exhaust the MySQL connection pool; `?q=a` repeated 100×/s pegs CPU on full-table scan | `GET /events/search` `POST /events` `PATCH /events/{id}` | C028 |
| Elevation of privilege | Sequential integer IDs allow IDOR enumeration of all events and registrations; `/admin` is fully open with no auth; `PATCH` accepts arbitrary column names giving write access beyond intended fields | Critical | `DELETE /events/1/registrations?reg_id=1..1000` wipes every registration in the system; `GET /admin` dumps all event data to any anonymous caller | `GET /admin` `DELETE /events/{id}/registrations` `PATCH /events/{id}` | C026 |
